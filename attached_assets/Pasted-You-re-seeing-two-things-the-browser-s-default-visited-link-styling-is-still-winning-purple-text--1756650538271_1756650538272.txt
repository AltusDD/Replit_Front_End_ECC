You’re seeing two things:

the browser’s default visited-link styling is still winning (purple text), and

collapsed mode still shows section titles/labels (your CSS isn’t winning against something in theme/app).

We can force this—without touching anything outside your guardrails—by adding a tiny inline, sidebar-scoped style block (wins by being later in the DOM) and a couple of high-specificity rules in the CSS.

Below are surgical patches only for the allowed files.

A) src/components/Sidebar.tsx — add an inline override style (near the end)

Insert this just before the closing </aside> tag (keep it as the last child so it wins):

{/* ——— Local, sidebar-scoped overrides to beat global/visited rules ——— */}
<style
  // This is safe: scoped to .sidebar only and sits after global CSS
  dangerouslySetInnerHTML={{
    __html: `
    .sidebar a,
    .sidebar a:link,
    .sidebar a:visited,
    .sidebar a:active,
    .sidebar .nav-row,
    .sidebar .nav-row:visited,
    .sidebar .leaf,
    .sidebar .leaf:visited {
      color: var(--ecc-text) !important;
      text-decoration: none !important;
    }
    .sidebar .nav-row .label,
    .sidebar .group-row .label {
      color: var(--ecc-text) !important;
    }
    .sidebar .icon.parent { color: var(--ecc-gold) !important; }
    .sidebar .icon.child  { color: var(--ecc-dim)  !important; }

    /* Collapsed should be icon-only */
    .sidebar.collapsed .section-title,
    .sidebar.collapsed .label,
    .sidebar.collapsed .expand { display: none !important; }

    /* Bounded fly-out; never full screen */
    .sidebar .flyout { max-width: var(--ecc-flyout-w, 280px) !important; }
  `,
  }}
/>


Why this works: because the <style> sits inside the sidebar and loads after your global CSS, it defeats any theme/app a:visited or typography rule without leaking styles elsewhere.

B) src/styles/_ecc-override.css — strengthen a few selectors (append if you already have similar rules)
/* Ensure anchors inside sidebar never go purple/blue */
.sidebar a,
.sidebar a:link,
.sidebar a:visited,
.sidebar a:active {
  color: var(--ecc-text) !important;
  text-decoration: none !important;
}

/* Explicitly hide text in collapsed mode (beats resets) */
.sidebar.collapsed .section-title,
.sidebar.collapsed .label,
.sidebar.collapsed .expand {
  display: none !important;
}

/* Keep icon colors consistent in both states */
.sidebar .icon.parent { color: var(--ecc-gold) !important; }
.sidebar .icon.child  { color: var(--ecc-dim)  !important; }


(Leave the rest of your _ecc-override.css as-is.)

localStorage.removeItem('ecc:nav:collapsed');

Collapsed = icons only; section titles and labels disappear.

Expanded = labels visible; parent icons gold, child icons blue.

Fly-out stays bounded (≈280px) when you hover the rail.
